# 🧠 Agent Skills 使用指南

> PromptAtelier 专业知识系统

## 什么是 Skills？

Skills 是可复用的专业知识模块，让 AI 在生成设计提示词时能够应用特定领域的专业知识。每个 Skill 包含：

- **触发词** - 自动匹配用户输入
- **专业知识** - Markdown 格式的专业内容
- **参考资料** - 相关文档和示例

## 快速开始

### 1. 查看已有 Skills

点击导航栏的 **🧠 专业技能** Tab，可以看到：

- 📊 统计面板：总数 / 已启用 / 可用
- 🔍 搜索框：按名称搜索
- 📋 Skills 卡片列表

### 2. 使用 Skills

**自动模式（推荐）**：
- 启用相关 Skills
- 正常使用"🎨 设计提示词"功能
- 系统会自动匹配并应用相关 Skills

**手动查看**：
- 点击 Skill 卡片的"查看详情"按钮
- 阅读专业知识内容

### 3. 管理 Skills

| 操作 | 说明 |
|------|------|
| 启用/禁用 | 点击卡片上的开关 |
| 查看详情 | 点击"查看详情"按钮 |
| 编辑 | 点击"✏️"编辑按钮 |
| 导出 | 点击"📤"导出按钮 |
| 删除 | 点击"🗑️"删除按钮 |

## 创建自定义 Skill

### 方式 1: 通过 UI 创建

1. 点击"**+ 创建 Skill**"按钮
2. 填写基本信息：
   - **ID**：唯一标识符（英文，如 `my-skill`）
   - **名称**：显示名称（如 `我的技能`）
   - **描述**：简要说明
3. 添加触发词和标签
4. 编写专业知识内容（Markdown 格式）
5. 点击"创建"保存

### 方式 2: 导入 JSON

1. 点击"**📥 导入 JSON**"按钮
2. 选择 JSON 文件
3. 系统自动创建 Skill

### 方式 3: 手动创建目录

```bash
skills/
└── my-custom-skill/
    ├── SKILL.md          # 必须：Skill 定义
    ├── assets/           # 可选：资源文件
    └── references/       # 可选：参考资料
```

## SKILL.md 格式

```markdown
---
name: "技能名称"
description: "技能描述"
version: "1.0.0"
triggers:
  - 触发词1
  - 触发词2
  - 触发词3
tags:
  - 标签1
  - 标签2
author: "作者名"
enabled: true
---

# 技能标题

这里是技能的主要内容，使用 Markdown 格式编写。

## 核心知识

详细的专业知识...

## 应用场景

- 场景1
- 场景2

## 最佳实践

1. 建议1
2. 建议2
```

## 工作原理

```
┌─────────────────────────────────────────────────────────┐
│                    用户输入设计需求                      │
└─────────────────────┬───────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────────┐
│          Skills 匹配引擎 (/api/skills/match)            │
│  - 扫描已启用的 Skills                                  │
│  - 匹配触发词和标签                                     │
│  - 返回相关 Skills 列表                                 │
└─────────────────────┬───────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────────┐
│         构建上下文 (/api/skills/context)                │
│  - 读取匹配 Skills 的完整内容                           │
│  - 组装专业知识上下文                                   │
└─────────────────────┬───────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────────┐
│                   注入系统提示词                         │
│  - 将 Skills 知识注入到 AI 系统提示词                   │
│  - AI 参考专业知识生成更好的提示词                      │
└─────────────────────┬───────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────────┐
│                     输出结果                             │
│  - 专业的设计提示词                                     │
│  - 显示应用的 Skills 列表                               │
└─────────────────────────────────────────────────────────┘
```

## API 参考

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/skills` | GET | 获取所有 Skills |
| `/api/skills/:id` | GET | 获取单个 Skill |
| `/api/skills` | POST | 创建新 Skill |
| `/api/skills/:id` | PUT | 更新 Skill |
| `/api/skills/:id` | DELETE | 删除 Skill |
| `/api/skills/:id/toggle` | PATCH | 切换启用状态 |
| `/api/skills/match` | POST | 匹配 Skills |
| `/api/skills/context` | POST | 获取匹配上下文 |
| `/api/skills/rescan` | POST | 重新扫描目录 |
| `/api/skills/:id/export` | GET | 导出 Skill |
| `/api/skills/import` | POST | 导入 Skill |

## 常见问题

### Q: Skills 不生效？

1. 确保 Skill 已启用（开关为绿色）
2. 检查触发词是否匹配用户输入
3. 确认 AI API 已配置

### Q: 如何分享 Skills？

1. 点击导出按钮，下载 JSON 文件
2. 将文件发送给他人
3. 对方通过导入功能添加

### Q: 自定义 Skill 不显示？

1. 确保 `SKILL.md` 文件存在
2. 检查 YAML frontmatter 格式
3. 点击"重新扫描"按钮刷新
