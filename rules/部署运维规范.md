# PromptAtelier 部署运维规范

## 1. 部署架构

### 1.1 简化部署架构
```
用户 → Nginx (可选) → Node.js App → 静态文件
                    ↓
                 JSON数据文件
```

### 1.2 服务器要求
- **操作系统**：Linux Ubuntu 20.04+ / Windows 10+ / macOS 10.15+
- **Node.js版本**：18.0.0+
- **内存要求**：最低 512MB，推荐 1GB
- **存储要求**：最低 100MB，推荐 500MB
- **网络要求**：公网IP（生产环境）

### 1.3 部署选项
1. **本地开发部署**：直接node启动
2. **生产环境部署**：PM2 + Nginx
3. **云服务部署**：Vercel/Netlify（前端） + Railway/Render（后端）

## 3. 部署方案

### 3.1 方案一：本地快速部署（开发测试）

**步骤**：
```bash
# 1. 克隆代码
git clone <repository-url>

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev

# 4. 访问应用
# 浏览器打开: http://localhost:3000
```

**验证**：
```bash

curl http://localhost:3000/api/health

# 测试API功能
curl -X POST http://localhost:3000/api/generate \
  -H "Content-Type: application/json" \
```

### 3.2 方案二：PM2生产部署（推荐）

**PM2配置文件**：
```javascript
// ecosystem.config.js
module.exports = {
  apps: [{
    name: 'prompt-atelier',
    script: './backend/server.js',
    instances: 1,
    exec_mode: 'fork',
    env: {
      NODE_ENV: 'development',

    out_file: './logs/out.log',
    log_file: './logs/combined.log',

    max_memory_restart: '200M'
  }]

```

**部署步骤**：
```bash
# 1. 安装PM2
npm install -g pm2

# 2. 克隆并安装
git clone <repository-url>
cd PromptAtelier
npm install --production

# 3. 启动PM2
pm2 start ecosystem.config.js --env production

# 4. 保存PM2配置
pm2 save
pm2 startup

# 5. 验证服务
pm2 status
pm2 logs prompt-atelier
```

### 3.3 方案三：Nginx反向代理（高并发）

**Nginx配置**：
```nginx
# /etc/nginx/sites-available/prompt-atelier
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    
    # 前端静态文件
    location / {
        root /var/www/prompt-atelier/frontend;
        try_files $uri $uri/ /index.html;
        
        # 缓存静态资源
        location ~* \.(css|js|png|jpg|jpeg|gif|ico|svg)$ {
            expires 1y;
            add_header Cache-Control "public, no-transform";
        }
    }
    
    # API代理
    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }
    
    # 健康检查
    location /health {
        proxy_pass http://localhost:8080/health;
    }
}
```

**启用配置**：
```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/prompt-atelier /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载Nginx
sudo systemctl reload nginx
```

### 4.2 日志配置
```javascript
// utils/logger.js
const fs = require('fs');
const path = require('path');

class Logger {
  constructor() {
    this.logDir = path.join(__dirname, '../logs');
    if (!fs.existsSync(this.logDir)) {
      fs.mkdirSync(this.logDir, { recursive: true });
    }
  }
  
  log(level, message, meta = {}) {
    const timestamp = new Date().toISOString();
    const logEntry = {
      timestamp,
      level,
      message,
      ...meta
    };
    
    // 控制台输出
    console.log(`[${timestamp}] ${level.toUpperCase()}: ${message}`);
    
    // 文件输出（生产环境）
    if (process.env.NODE_ENV === 'production') {
      const logFile = path.join(this.logDir, `${level}.log`);
      fs.appendFileSync(logFile, JSON.stringify(logEntry) + '\n');
    }
  }
  
  info(message, meta) { this.log('info', message, meta); }
  error(message, meta) { this.log('error', message, meta); }
  debug(message, meta) { this.log('debug', message, meta); }
}

module.exports = new Logger();
```

### 4.3 性能监控
```javascript
// middleware/monitoring.js
const logger = require('../utils/logger');

function monitorPerformance(req, res, next) {
  const startTime = Date.now();
  
  res.on('finish', () => {
    const duration = Date.now() - startTime;
    logger.info('API Request', {
      method: req.method,
      url: req.url,
      statusCode: res.statusCode,
      duration: `${duration}ms`,
      userAgent: req.get('User-Agent')
    });
    
    // 慢查询警告
    if (duration > 3000) {
      logger.error('Slow API Response', {
        method: req.method,
        url: req.url,
        duration: `${duration}ms`
      });
    }
  });
  
  next();
}

module.exports = monitorPerformance;
```

### 4.4 监控脚本
```bash
#!/bin/bash
# monitor.sh - 服务监控脚本

# 检查服务状态
check_service() {
    local url="http://localhost:8080/health"
    local response=$(curl -s -o /dev/null -w "%{http_code}" $url)
    
    if [ $response -eq 200 ]; then
        echo "✅ 服务正常运行"
    else
        echo "❌ 服务异常，状态码: $response"
        # 发送告警通知
        notify_admin "PromptAtelier服务异常"
    fi
}

# 检查磁盘空间
check_disk() {
    local usage=$(df / | awk 'NR==2 {print $5}' | sed 's/%//')
    if [ $usage -gt 80 ]; then
        echo "⚠️  磁盘使用率过高: ${usage}%"
        notify_admin "磁盘空间不足"
    fi
}

# 检查内存使用
check_memory() {
    local mem_usage=$(free | awk 'NR==2{printf "%.2f", $3*100/$2}')
    if (( $(echo "$mem_usage > 80" | bc -l) )); then
        echo "⚠️  内存使用率过高: ${mem_usage}%"
    fi
}

# 告警通知（可集成企业微信、钉钉等）
notify_admin() {
    local message=$1
    # echo "告警: $message" | mail -s "PromptAtelier监控告警" admin@yourdomain.com
    echo "$(date): $message" >> /var/log/prompt-atelier-alerts.log
}

# 执行检查
echo "开始系统监控检查..."
check_service
check_disk
check_memory
echo "监控检查完成"
```

## 5. 故障处理

### 5.1 常见问题诊断

**问题1：服务无法启动**
```bash
# 检查端口占用
netstat -tulpn | grep :8080

# 检查Node.js版本
node --version

# 检查依赖
npm ls --depth=0

# 查看详细错误
NODE_ENV=development node backend/server.js
```

**问题2：API响应慢**
```bash
# 检查系统资源
top
free -h
df -h

# 查看PM2进程状态
pm2 monit

# 查看应用日志
pm2 logs prompt-atelier --lines 100
```

**问题3：前端页面无法加载**
```bash
# 检查Nginx状态
sudo systemctl status nginx

# 检查Nginx配置
sudo nginx -t

# 查看Nginx错误日志
sudo tail -f /var/log/nginx/error.log
```

### 5.2 故障恢复流程

**步骤1：问题识别**
- 检查监控告警
- 查看应用日志
- 验证服务状态

**步骤2：快速恢复**
```bash
# 重启应用
pm2 restart prompt-atelier

# 重启Nginx
sudo systemctl restart nginx

# 检查恢复状态
curl http://localhost:8080/health
```

**步骤3：问题分析**
- 分析日志文件
- 检查系统资源
- 识别根本原因

**步骤4：预防措施**
- 更新监控规则
- 优化代码逻辑
- 调整系统配置

### 5.3 备份恢复

**数据备份脚本**：
```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/backup/prompt-atelier"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份代码
tar -czf $BACKUP_DIR/code_$DATE.tar.gz /var/www/prompt-atelier

# 备份数据文件
cp -r /var/www/prompt-atelier/data $BACKUP_DIR/data_$DATE

# 备份配置文件
cp /etc/nginx/sites-available/prompt-atelier $BACKUP_DIR/nginx_$DATE.conf

# 清理旧备份（保留7天）
find $BACKUP_DIR -name "*.tar.gz" -mtime +7 -delete

echo "备份完成: $BACKUP_DIR"
```

**恢复脚本**：
```bash
#!/bin/bash
# restore.sh

BACKUP_FILE=$1
RESTORE_DIR="/var/www/prompt-atelier"

if [ -z "$BACKUP_FILE" ]; then
    echo "用法: ./restore.sh <备份文件>"
    exit 1
fi

# 停止服务
pm2 stop prompt-atelier

# 恢复代码
tar -xzf $BACKUP_FILE -C /

# 重新安装依赖
cd $RESTORE_DIR && npm install --production

# 启动服务
pm2 start prompt-atelier

echo "恢复完成"
```

## 6. 安全配置

### 6.1 基础安全配置
```javascript
// middleware/security.js
const helmet = require('helmet');
const rateLimit = require('express-rate-limit');

// 安全头设置
app.use(helmet({
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      styleSrc: ["'self'", "'unsafe-inline'", "https://cdn.tailwindcss.com"],
      scriptSrc: ["'self'"],
      imgSrc: ["'self'", "data:", "https:"]
    }
  }
}));

// 限流配置
const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15分钟
  max: 100, // 每IP最多100个请求
  message: '请求过于频繁，请稍后再试'
});

app.use('/api/', limiter);
```
